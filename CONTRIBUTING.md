# Contributing to Hirenix

First off, thank you for considering contributing to Hirenix! It's people like you that make Hirenix such a great tool for career acceleration. We welcome contributions from everyone, whether it's filing bug reports, proposing new features, or submitting pull requests.

## How Can I Contribute?

### üêõ Reporting Bugs

Before creating bug reports, please check the issue tracker to see if the problem has already been reported. If it hasn't, feel free to open a new issue.

When reporting a bug, please include:
- A clear and descriptive title.
- Steps to reproduce the bug.
- The expected behavior vs. the actual behavior.
- Details about your environment (OS, browser, Node/Python version).
- Screenshots or error logs if applicable.

### üí° Suggesting Enhancements

Have an idea for a new feature or improvement? We'd love to hear it! Open an issue on GitHub and provide:
- A clear and descriptive title.
- A detailed explanation of the proposed feature.
- Why this feature would be useful to the community.
- Any relevant examples or mockups.

### üíª Contributing Code

If you want to contribute code to fix a bug or add a feature, please follow these steps:

1. **Fork the Repository**: Click the "Fork" button at the top right of the repository page.
2. **Clone your Fork**:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Hirenix.git
   cd Hirenix
   ```
3. **Create a Branch**: Create a new branch for your changes. Use a descriptive name like `fix/login-bug` or `feature/new-ai-model`.
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your Changes**: Implement your changes in the codebase. Ensure your code follows the project's style guidelines.
5. **Test your Changes**: Run the project locally and verify that your changes work as expected and do not break existing functionality.
6. **Commit your Changes**: Write clear and concise commit messages.
   ```bash
   git commit -m "Add short description of the feature/fix"
   ```
7. **Push to your Fork**:
   ```bash
   git push origin feature/your-feature-name
   ```
8. **Submit a Pull Request**: Go to the original Hirenix repository and open a Pull Request against the `main` branch. Provide a detailed description of your changes in the PR.

## Development Setup

Please refer to the `README.md` files in the root, `frontend`, and `backend` directories for specific instructions on setting up the development environment.

## Code Style

- **Frontend**: We use Prettier and ESLint for the Next.js frontend. Please ensure your code passes linting before submitting a PR.
- **Backend**: We use standard PEP 8 guidelines for Python. Try to keep your code clean and well-documented.

## Questions?

If you have any questions or need help with your contribution, feel free to reach out via GitHub issues or discussions.
